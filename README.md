## Описание программы

**Expense Tracker** — это десктопное GUI‑приложение для учета личных финансов. Программа позволяет: 
* Добавлять записи о расходах с указанием суммы, категории и даты.
* Просматривать историю трат в виде удобной таблицы.
* Фильтровать расходы по конкретной категории или за выбранный период дат.
* Автоматически подсчитывать общую сумму расходов для отображаемого списка.
* Автоматически сохранять данные в формате JSON (и загружать их при запуске).
* Валидировать ввод (проверка на положительные числа и корректный формат даты).

## Требования для запуска

Программа использует исключительно стандартную библиотеку Python, поэтому установка сторонних пакетов через `pip` **не требуется**.

* Установленный **Python версии 3.6 или выше**.
* Встроенная библиотека `tkinter` (поставляется вместе с Python).

## Как запустить

Откройте терминал (командную строку) и выполните следующие команды:

```bash

git clone [https://github.com/Renic228/Expense-Tracker.git](https://github.com/Renic228/Expense-Tracker.git)

# 2. Переход в директорию проекта
cd Expense-Tracker

# 3. Запуск приложения
python main.py
```

---

## Как пользоваться

1. **Добавление расхода:** В верхней панели введите сумму (положительное число), выберите категорию из выпадающего списка (или впишите свою) и укажите дату в формате `ГГГГ-ММ-ДД`.
2. Нажмите кнопку **«Добавить»**. Расход появится в таблице, а данные автоматически сохранятся.
3. **Фильтрация:** В средней панели выберите нужную категорию и/или укажите диапазон дат (поля «С» и «По»).
4. Нажмите **«Применить»**, чтобы отфильтровать таблицу и пересчитать общую сумму.
5. Нажмите **«Сбросить»**, чтобы вернуть отображение всех расходов.

## Формат хранения данных (JSON)

Данные хранятся локально в файле `expenses.json` в виде массива объектов (списка словарей). Пример структуры:

```json
[
    {
        "date": "2026-04-20",
        "category": "Еда",
        "amount": 1250.50
    },
    {
        "date": "2026-04-21",
        "category": "Транспорт",
        "amount": 150.00
    }
]
```

## Тестирование

| Тип теста | Сценарий (Действие) | Ожидаемый результат |
| :--- | :--- | :--- |
| **Позитивный** | Заполнить все поля корректно (Сумма: 500, Категория: Еда, Дата: 2026-04-20) и нажать «Добавить» | Запись появляется в таблице, поля ввода очищаются, общая сумма пересчитывается, файл `expenses.json` обновляется. |
| **Негативный (Сумма)** | Ввести в поле «Сумма» текст (например, `abc`) и нажать «Добавить» | Появление всплывающего окна с ошибкой: «Сумма должна быть числом.» Запись не добавляется. |
| **Негативный (Сумма)** | Ввести в поле «Сумма» отрицательное число (`-100`) и нажать «Добавить» | Появление всплывающего окна с ошибкой: «Сумма должна быть положительным числом.» |
| **Негативный (Дата)** | Ввести дату в неверном формате (например, `20.04.2026` вместо `2026-04-20`) | Ошибка: «Дата должна быть в формате ГГГГ-ММ-ДД.» |
| **Фильтрация (Категория)** | Выбрать в фильтре категорию «Транспорт» и нажать «Применить» | В таблице остаются только записи с категорией «Транспорт», итоговая сумма пересчитывается только для них. |
| **Сброс фильтров** | Нажать кнопку «Сбросить» после применения любых фильтров | В таблице снова отображаются абсолютно все записи из базы данных. |
